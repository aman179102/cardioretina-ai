# Dataset Format Guide

## Overview

This guide describes the expected format for training data used by the CardioRetina-AI pipeline.

## CSV Structure

The training pipeline expects a CSV file with the following required columns:

### Required Columns

| Column | Type | Description | Example |
|--------|------|-------------|---------|
| `image_path` | string | Relative path to retinal image | `images/001_left.jpg` |
| `label` | integer | Binary classification (0=low risk, 1=high risk) | `1` |
| `age` | float | Patient age in years | `55.0` |
| `systolic_bp` | float | Systolic blood pressure in mmHg | `140.0` |
| `diastolic_bp` | float | Diastolic blood pressure in mmHg | `90.0` |
| `cholesterol` | float | Total cholesterol in mg/dL | `240.0` |
| `bmi` | float | Body Mass Index | `28.5` |
| `smoking` | integer | Smoking status (0=No, 1=Yes) | `0` |
| `diabetes` | integer | Diabetes status (0=No, 1=Yes) | `0` |
| `physical_activity` | integer | Activity level (0=Sedentary, 1=Active) | `1` |

### Sample Row

```csv
image_path,label,age,systolic_bp,diastolic_bp,cholesterol,bmi,smoking,diabetes,physical_activity
images/patient_001.jpg,1,55.0,140.0,90.0,240.0,28.5,0,0,1
```

## Image Requirements

### Format
- **Accepted formats**: JPEG (.jpg, .jpeg), PNG (.png), BMP (.bmp), TIFF (.tif, .tiff)
- **Color mode**: RGB (color fundus images)
- **Preprocessing**: Automatic resize to 224×224, CLAHE enhancement applied

### Quality Guidelines
- **Resolution**: Minimum 512×512 pixels (higher is better)
- **Focus**: Sharp image with clear vessel details
- **Field of view**: Standard 45° fundus view
- **Artifacts**: Minimal blur, glare, or motion artifacts

### File Organization

Recommended directory structure:

```
dataset/
├── data.csv              # Main metadata file
├── images/
│   ├── patient_001.jpg
│   ├── patient_002.jpg
│   └── ...
└── splits/               # Generated by split script
    ├── train.csv
    ├── val.csv
    └── test.csv
```

## Data Validation

### Automatic Validation

Run the validation script to check data integrity:

```bash
python -m cardioretina.data.validate data.csv --image-dir ./images/
```

### Validation Checks

The validator performs:

1. **CSV Structure**
   - All required columns present
   - No duplicate rows
   - Data types correct

2. **Image Files**
   - All referenced images exist
   - Images are readable
   - Format is supported

3. **Value Ranges**
   - Age: 0-120 years
   - Systolic BP: 50-300 mmHg
   - Diastolic BP: 30-200 mmHg
   - Cholesterol: 50-600 mg/dL
   - BMI: 10-70 kg/m²
   - Binary fields: 0 or 1 only

4. **Label Distribution**
   - At least one sample per class
   - Class imbalance warnings if applicable

## Dataset Splitting

### Create Train/Val/Test Splits

```bash
python -m cardioretina.data.split data.csv --output-dir ./splits/ --train-ratio 0.7 --val-ratio 0.15
```

### Default Split Ratios

| Split | Ratio | Purpose |
|-------|-------|---------|
| Training | 70% | Model training |
| Validation | 15% | Hyperparameter tuning, early stopping |
| Test | 15% | Final evaluation (held out) |

### Stratification

The splitter maintains class distribution across splits:
- Preserves approximately same % of positive/negative cases
- Prevents data leakage
- Ensures representative evaluation

## Data Preprocessing

### Automatic Preprocessing

During training, the pipeline automatically applies:

1. **CLAHE Enhancement**
   - Improves vessel visibility
   - Adaptive histogram equalization

2. **Normalization**
   - ImageNet statistics for pre-trained models
   - Per-channel mean/std normalization

3. **Augmentation** (training only)
   - Random horizontal/vertical flip
   - Random rotation (±15°)
   - Color jitter
   - Random crop

### Manual Preprocessing

For custom preprocessing:

```python
from cardioretina.data.preprocessing import preprocess_image

processed = preprocess_image(
    image_path,
    target_size=(224, 224),
    apply_clahe=True,
    normalize=True
)
```

## Data Privacy

### Best Practices

1. **Anonymization**
   - Remove patient names, IDs, dates from filenames
   - Use generic identifiers (patient_001, etc.)

2. **Storage**
   - Encrypt at rest if containing real patient data
   - Follow institutional data governance policies

3. **Transmission**
   - Use secure channels for data transfer
   - Never share identifiable patient images

4. **Compliance**
   - HIPAA (US) or GDPR (EU) as applicable
   - Obtain proper ethics approvals for research use

## Troubleshooting

### Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| "Image not found" | Wrong image_dir path | Check relative path from CSV |
| "Invalid label" | Non-binary values | Ensure labels are 0 or 1 |
| "Missing column" | Column name typo | Verify column headers match exactly |
| "Out of range" | Data entry errors | Check data collection pipeline |
